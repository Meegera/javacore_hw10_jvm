public class JvmComprehension {

    public static void main(String[] args) {
        int i = 1;                      // 1
        Object o = new Object();        // 2
        Integer ii = 2;                 // 3
        printAll(o, i, ii);             // 4
        System.out.println("finished"); // 7
    }

    private static void printAll(Object o, int i, Integer ii) {
        Integer uselessVar = 700;                   // 5
        System.out.println(o.toString() + i + ii);  // 6
    }
}


/*
Описание:
В начале выполнения main-в стеке появится фрейм main
1) В стеке - в main - создастся переменная i и инициализируется со значением = 1
2) В стеке - в main - создастся переменная o, в heap выделится память под объект Object. Переменная o будет содержать ссылку на объект Object из кучи
3)В стеке - в main - создастся переменная ii, в heap выделится память под объект Integer = 2. Переменная ii будет содержать ссылку на объект Integer из кучи
4) В стеке создастся новый фрейм printAll. Будет вызвана данная функция.
5)В стеке - в printAll - создастся переменная uselessVar, в heap выделится память под объект Integer = 700. Переменная uselessVar будет содержать ссылку на объект Integer из кучи
6)В стеке создастся новый фрейм, который будет содержать временную строку, полученную с помощью конкатенации
7)В стеке создастся новый фрейм, куда будет передана ссылка на объект "finished", который будет располагаться в куче

Application ClassLoader загрузил классы: main и printAll, инициализировал их перед началом выполнения программы

По окончанию выполнения метода main, все объекты, созданные в куче, становятся недостижимыми, тк нет ссылок на них. Сборщик мусора позаботиться о высвобождении памяти, занятой этими объектами, когда это будет нужно.
*/
